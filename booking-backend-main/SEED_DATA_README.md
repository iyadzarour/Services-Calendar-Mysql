## كيفية استخدام البيانات

### تشغيل ملف Seed Data

```bash
# تشغيل ملف seed_data.sql
mysql -u root booking_db < seed_data.sql

# أو في PowerShell:
Get-Content seed_data.sql | mysql -u root booking_db
```

### تسجيل الدخول كمدير
- البريد الإلكتروني: `admin@example.com`
- كلمة المرور: `admin123`

### تسجيل الدخول كسكرتارية
- البريد الإلكتروني: `secretary@example.com`
- كلمة المرور: `secret123`

### تسجيل الدخول كموظف
- البريد الإلكتروني: `employee1@example.com` أو `employee2@example.com`
- كلمة المرور: `emp123`

### تسجيل الدخول كعميل
- البريد الإلكتروني: أي بريد من قائمة العملاء (مثل `ahmed@customer.com`)
- كلمة المرور: `user123`

## ملاحظات مهمة

1. **كلمات المرور:** جميع كلمات المرور مشفرة باستخدام bcrypt
2. **إعادة تعيين البيانات:** لإعادة تعيين البيانات، قم بتشغيل `mysql_schema.sql` أولاً ثم `seed_data.sql`
3. **البيانات الحقيقية:** في بيئة الإنتاج، قم بحذف هذا الملف واستخدام بيانات حقيقية

## الحقول الاختيارية

بعض الحقول مثل `district`, `lat`, `lng` للعملاء لم يتم ملؤها في البيانات الوهمية، يمكنك إضافتها يدوياً لاختبار ميزة Location-Aware Scheduling.
